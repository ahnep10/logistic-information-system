## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Developer filesystem → PostgreSQL | DATABASE_URL is the trust boundary; if exposed, attacker has full DB access |
| .env file → Git | .env must never be committed; AUTH_SECRET and DATABASE_URL are secrets |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-1-03 | Spoofing | AUTH_SECRET missing from .env | high | mitigate | Auth.js throws at startup if AUTH_SECRET is absent — fail-fast prevents insecure operation. Ensure .env is populated before `npm run dev`. |
| T-1-04 | Information Disclosure | .env committed to git | high | mitigate | create-next-app adds .env* to .gitignore by default. Verify with `git status` before first commit. Task 1 Step 4 explicitly checks this. |
| T-1-ENV | Tampering | DATABASE_URL pointing to production DB during development | medium | accept | Development environment risk — developer is responsible for pointing DATABASE_URL at a dev/local database, not a shared production DB. Document in .env.example. |
</threat_model>
