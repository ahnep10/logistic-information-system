## Screen Inventory

### Screen 1: Login Page (`/login`)

**Route:** `/login` — unauthenticated access only. Redirect to role home if already authenticated.

**Focal point:** Sign in button — full-width accent (`bg-blue-600`), anchored at the bottom of the centered Card. The eye naturally lands here after scanning the two input fields above it.

**Layout:** Full-screen, `min-h-screen flex items-center justify-center bg-zinc-50`. No sidebar.

**Structure:**
```
[bg-zinc-50 full screen]
  └── Card (max-w-sm, centered)
        ├── CardHeader
        │     ├── "Logistics MIS"  [24px semibold, text-center]
        │     └── "Sign in to your account"  [14px, text-zinc-500, text-center]
        └── CardContent
              ├── Email input (label: "Email", type: email, autocomplete: email)
              ├── Password input (label: "Password", type: password, autocomplete: current-password)
              ├── Server error area  [12px, text-red-600 — shown on invalid credentials]
              └── Button "Sign in"  [full width, accent]
```

**Interaction contract:**
- Sign in button shows Loader2 spinner (`animate-spin w-4 h-4 mr-2`) while submitting, becomes `disabled`
- On credential failure: render `"Invalid email or password. Please check your credentials."` as `FormMessage` below the password field (not a toast)
- On success: middleware post-login redirect — MANAGER → `/dashboard`, STAFF → `/inventory`
- No "Forgot password" link (no email features in Phase 1 — confirmed in CONTEXT.md)
- No "Create account" link (no self-registration — D-03 in CONTEXT.md)
- Tab order: Email → Password → Sign in button

## Interaction Contracts

### Auth guard behavior (non-visual but defines redirect UX)

| Scenario | System Response |
|----------|----------------|
| Unauthenticated user accesses any protected route | Redirect to `/login` — no error message |
| STAFF user accesses `/dashboard`, `/reports`, or `/users` | Middleware 403 redirect to `/inventory` — no error page, no toast (D-09) |
| Authenticated user accesses `/login` | Redirect to role home — MANAGER → `/dashboard`, STAFF → `/inventory` |
| Session expires | Next request returns 401, middleware redirects to `/login` |

### Form submission pattern (all forms in Phase 1)

1. User submits form
2. Client-side Zod validation runs — invalid fields show `FormMessage` inline
3. If client valid: button enters Loading state, Server Action invoked
4. Server re-validates with same Zod schema
5. On server error: return error object, render as `FormMessage` on relevant field
6. On success: close Dialog (if in Dialog) or navigate away

### Sidebar role rendering (no runtime toggling)

The sidebar nav item list is determined at render time from `session.user.role`. There is no toggle, no animation, no deferred load. STAFF simply never sees `<DashboardNavItem />`, `<ReportsNavItem />`, or `<UsersNavItem />` — they are conditionally excluded from the JSX, not hidden with CSS.

## Accessibility

| Requirement | Implementation |
|-------------|---------------|
| Form input labels | shadcn `Label` with `htmlFor` matching `Input` `id` — enforced by shadcn `FormField` |
| Field error association | `aria-describedby` pointing to `FormMessage` element id |
| Deactivate confirmation | Use `AlertDialog` (not plain `Dialog`) — renders as `role="alertdialog"` |
| Active nav item | `aria-current="page"` on the current route's anchor element |
| Icon-only buttons (table actions) | `aria-label="Edit user"` / `aria-label="Deactivate user"` on `Button size="icon"` |
| Keyboard navigation — sidebar | Radix primitives handle full keyboard accessibility natively |
| Keyboard navigation — dialogs | Radix `Dialog` traps focus; Escape key closes; focus returns to trigger on close |
| Color contrast — sidebar text | `text-slate-100` on `bg-slate-900`: ratio ≈ 16:1 (exceeds WCAG AAA) |
| Color contrast — body text | `text-zinc-900` on `bg-white`: ratio ≈ 18:1 |
| Color contrast — muted text | `text-zinc-500` on `bg-white`: ratio ≈ 4.6:1 (meets WCAG AA) |
| Focus rings | `ring-2 ring-blue-600 ring-offset-2` via shadcn default focus styles |
| Touch targets | All interactive elements minimum 40px height (`h-10`) |
