## Deviations from Plan

### Auto-handled Ecosystem Changes

**1. [Rule 2 - Compatibility] shadcn/ui v4 uses base-nova style, not new-york**
- **Found during:** Task 1 (shadcn init)
- **Issue:** shadcn/ui v4 removed the "new-york" and "default" style system, replacing with "base-nova". The `--style new-york` option returned an error.
- **Fix:** Accepted "base-nova" style (v4 default). Visual design defaults differ but all component APIs and functionality are identical. The UI-SPEC.md's "New York" preference was a v3-era choice.
- **Impact:** Cosmetic only — no functional difference for the MIS use case.

**2. [Rule 2 - Compatibility] Form component not in shadcn/ui v4 registry**
- **Found during:** Task 1 (shadcn add form)
- **Issue:** `npx shadcn@latest add form` returned no output — Form is not in the v4 registry. shadcn v4 switched from `@radix-ui` to `@base-ui/react` primitives; the form wrapper wasn't migrated.
- **Fix:** Created `components/ui/form.tsx` manually using the same react-hook-form wrapping pattern as shadcn/ui v3, but without `@radix-ui/react-label` or `@radix-ui/react-slot` dependencies (replaced with native React `cloneElement` and Label from shadcn v4).
- **Files modified:** `components/ui/form.tsx` (created)

**3. [Rule 3 - Blocking] prisma.config.ts auto-generated by prisma init removed**
- **Found during:** Task 1 (prisma init)
- **Issue:** Prisma 6.19.3's `prisma init` created a `prisma.config.ts` that imports `dotenv` (not installed) and uses a different generator path. This file was not expected by the plan and would break the seed setup.
- **Fix:** Deleted `prisma.config.ts`. The `schema.prisma` + `package.json` seed config is the canonical Prisma 6 approach per RESEARCH.md. The `package.json#prisma.seed` deprecation warning from Prisma 6.19.3 is noted — this is for Prisma 7 migration only.
- **Files modified:** Deleted `prisma.config.ts`

**4. [Rule 2 - Compatibility] zod 4.x and @hookform/resolvers 5.x installed**
- **Found during:** Task 1 (npm install)
- **Issue:** `npm install zod` installed v4.4.3 (not v3.x). `npm install @hookform/resolvers` installed v5.4.0 (not v3.x). Both are the current npm latest.
- **Fix:** Accepted newer versions. Zod 4 has a backward-compatible API for all usage patterns in this plan (`z.object`, `z.string().email()`, `z.enum`, `z.infer`, `safeParse`). @hookform/resolvers 5.x explicitly supports Zod 4.
- **Impact:** None — all patterns work identically with v4 Zod.
