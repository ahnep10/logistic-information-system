## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Browser → Server Action | recordStockIn / recordStockOut callable by any request; session check is the auth gate |
| Server Action → DB | prisma.$transaction wraps both product update and transaction create; DB CHECK is the hard floor |
| migration.sql → DB | Custom SQL appended to Prisma-generated migration — incorrect SQL would apply to prod DB |
