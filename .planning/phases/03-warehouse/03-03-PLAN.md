IMPORTS: import { Suspense } from "react"; import { prisma } from "@/lib/prisma"; import { Prisma } from "@prisma/client"; import InventoryClient from "./inventory-client"

TYPE DEFINITION (inline): type SearchParams = { productId?: string; from?: string; to?: string; type?: string }; type Props = { searchParams: Promise<SearchParams> }

EXPORT DEFAULT async function InventoryPage({ searchParams }: Props):

Step 1 — Await searchParams. This is a Next.js 15 REQUIRED step. Access const params = await searchParams. Do NOT access searchParams.productId or any property before awaiting — that returns undefined silently (RESEARCH.md Pitfall 1).

Step 2 — Build Prisma where clause as Prisma.StockTransactionWhereInput = {}:
- If params.productId is truthy: set where.productId = params.productId
- If params.type is "STOCK_IN": set where.type = "STOCK_IN"
- If params.type is "STOCK_OUT": set where.type = "STOCK_OUT"
- Date range: if params.from or params.to are present, build where.createdAt = {} and:
  - If params.from: set gte = new Date(`${params.from}T00:00:00.000Z`)
  - If params.to: set lte = new Date(`${params.to}T23:59:59.999Z`) — MUST use T23:59:59.999Z (not T00:00:00) to include all records on the to date per RESEARCH.md Pitfall 4
- If neither params.from nor params.to: apply default 30-day window: const thirtyDaysAgo = new Date(); thirtyDaysAgo.setDate(thirtyDaysAgo.getDate() - 30); set where.createdAt = { gte: thirtyDaysAgo } per D-15

Step 3 — Parallel fetch with Promise.all:
- prisma.stockTransaction.findMany with where, orderBy: { createdAt: "desc" }, take: 200, include: { product: { select: { id: true, name: true, sku: true } }, createdBy: { select: { name: true } } }
- prisma.product.findMany with where: { isActive: true }, orderBy: { name: "asc" }, select: { id: true, name: true }

Step 4 — Return JSX: wrap InventoryClient in Suspense per RESEARCH.md Pitfall 2 (required because InventoryClient uses useSearchParams). Suspense fallback is a div with className="p-6 text-sm text-muted-foreground" and text "Loading...". InventoryClient receives props: transactions={transactions}, products={products}, currentParams={params}.

The page header ("Inventory History" h1) is rendered OUTSIDE the Suspense boundary so it appears immediately during loading.

Structure:
div wrapper > h1 "Inventory History" (text-2xl font-semibold mb-6) > Suspense > InventoryClient.
