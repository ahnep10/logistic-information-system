## Phase Goal

**As a** Manager, **I want to** generate inventory, stock-movement, and purchase-order reports and export any of them as an Excel (.xlsx) file, **so that** I can share and further analyze operational data offline.

(Source: ROADMAP.md Phase 6 Goal line, restated in user-story form per MVP_MODE — no capability added beyond what ROADMAP already states.)

<objective>
Add the Excel-export capability (REPT-04) this codebase's reports currently lack: one Route Handler per report type (`/api/reports/inventory`, `/api/reports/movements`, `/api/reports/purchase-orders`), each independently re-deriving and re-running its own Prisma query, converting the result into an `.xlsx` workbook via the `xlsx` (SheetJS) package, and streaming it back with the correct OOXML headers. This is this codebase's first-ever binary-streaming Route Handler and its first new npm dependency since Phase 5's `recharts`.

Purpose: Close the loop on D-04/D-05/D-06 — every report a manager can view must also be downloadable as a real spreadsheet reflecting exactly what's on screen, gated behind an explicit human-verify checkpoint for the `xlsx` package's known frozen/CVE status.
Output: `lib/utils/route-auth.ts`, `app/api/reports/inventory/route.ts`, `app/api/reports/movements/route.ts`, `app/api/reports/purchase-orders/route.ts`, `tests/reports-export.test.ts`, `xlsx` added to `package.json`.
</objective>

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| npm registry -> project dependencies | `npm install xlsx` (CDN tarball or frozen npm build, per Task 0's approved path) pulls third-party code into `node_modules`; that code executes inside this app's build and runtime process |

| Route Handler -> PostgreSQL via Prisma | Each handler independently re-derives its query from `new URL(request.url).searchParams`; only validated date ranges reach any `where` clause |
