## Phase Goal

**As a** Manager, **I want to** generate inventory, stock-movement, and purchase-order reports and export any of them as an Excel (.xlsx) file, **so that** I can share and further analyze operational data offline.

(Source: ROADMAP.md Phase 6 Goal line, restated in user-story form per MVP_MODE — no capability added beyond what ROADMAP already states.)

<objective>
Replace the Phase 1 stub at `app/(protected)/reports/page.tsx` with a single Tabs-driven `/reports` page (D-01 — one page, Tabs selector for the three report types) covering all three report types this phase requires: Inventory (REPT-01), Movements (REPT-02), and Purchase Orders (REPT-03). Each report type is delivered as a complete vertical slice (Prisma query -> validated prop -> rendered table) in its own task, reusing existing severity/status badge helpers and query shapes from `/products`, `/inventory`, and `/purchase-orders` verbatim.

Purpose: Give managers a single place to generate the three operational reports the roadmap requires, with the export capability (REPT-04, Plan 06-02) hanging off the same page via plain `<a href download>` links.
Output: `lib/utils/reports.ts`, `app/(protected)/reports/page.tsx`, `app/(protected)/reports/reports-client.tsx`, `tests/reports.test.ts`.
</objective>

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Browser URL bar -> `/reports?type=` | Any string value is attacker/user-controllable and reaches the Server Component as raw text |
| Browser URL bar -> `/reports?from=`/`?to=` (movements only) | Any string value is attacker/user-controllable; must be regex-validated before constructing a `Date` |
| `reports/page.tsx` -> PostgreSQL via Prisma | Only a validated `ReportType` and a validated `{gte,lte}` Date pair ever reach a `where` clause — raw strings never cross this boundary |
