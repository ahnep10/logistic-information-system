Product model fields: id String @id @default(cuid()), name String, sku String @unique, categoryId String, category Category @relation(fields: [categoryId], references: [id]), reorderThreshold Int @default(0), currentStock Int @default(0), isActive Boolean @default(true), createdAt DateTime @default(now()), updatedAt DateTime @updatedAt. Supplier model fields: id String @id @default(cuid()), name String, contactPerson String, phone String, email String, address String, isActive Boolean @default(true), createdAt DateTime @default(now()), updatedAt DateTime @updatedAt. Map to @@map("suppliers"). All contact fields (contactPerson, phone, email, address) are required non-nullable Strings — no ? suffix. This is enforced by Zod on form submission, not by DB nullable.

Per D-04: currentStock Int @default(0) is required on Product. Per D-05: no initialStock or quantity field should exist on the Product model. Per D-03: Category.name has @unique at the DB level. Per D-02: no onDelete cascade configuration — use Prisma default (Restrict), allowing soft-deactivation of categories without orphaning products.

This command: (1) connects to the DATABASE_URL in .env, (2) creates the categories, products, and suppliers tables with FK constraint from products.category_id → categories.id, (3) runs prisma generate to regenerate @prisma/client with the new model types.
