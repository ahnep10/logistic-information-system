createSupplier(formData: FormData) — export async function:
1. Call await requireManager()
2. Run createSupplierSchema.safeParse({ name: formData.get("name"), contactPerson: formData.get("contactPerson"), phone: formData.get("phone"), email: formData.get("email"), address: formData.get("address") })
3. If !parsed.success return { error: "Invalid input. Please check all fields." }
4. Create: await prisma.supplier.create({ data: { name: parsed.data.name, contactPerson: parsed.data.contactPerson, phone: parsed.data.phone, email: parsed.data.email, address: parsed.data.address } })
5. Call revalidatePath("/suppliers")
6. Return { success: true }

No uniqueness pre-check for suppliers — supplier name is not unique (multiple suppliers can share a company name, e.g. different regional offices). Email is also not unique per the schema design in RESEARCH.md.

updateSupplier(formData: FormData) — export async function:
1. Call await requireManager()
2. Run updateSupplierSchema.safeParse({ id: formData.get("id"), name: formData.get("name"), contactPerson: formData.get("contactPerson"), phone: formData.get("phone"), email: formData.get("email"), address: formData.get("address") })
3. If !parsed.success return { error: "Invalid input. Please check all fields." }
4. Update: await prisma.supplier.update({ where: { id: parsed.data.id }, data: { name: parsed.data.name, contactPerson: parsed.data.contactPerson, phone: parsed.data.phone, email: parsed.data.email, address: parsed.data.address } })
5. Call revalidatePath("/suppliers")
6. Return { success: true }

toggleSupplierActive(id: string, isActive: boolean) — export async function:
1. Call await requireManager()
2. await prisma.supplier.update({ where: { id }, data: { isActive } })
3. Call revalidatePath("/suppliers")
4. Return { success: true }

Do NOT call prisma.supplier.delete() anywhere — soft-delete only per SUPL-03. All three mutations use requireManager() — this implements the CONTEXT.md decision overriding REQUIREMENTS.md "Staff" wording for SUPL-01/02/03.
